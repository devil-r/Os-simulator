Introduction

  • Developers : Sushruth V

Page replacement algorithms are the algorithms based on which the Operating System decides which page to replace from the main memory with the page requested by the CPU. Page Replacement has to be done because of the undeniable fact that the physical main memory is smaller that the virtual memory. Whenever the CPU requests for a page that is not available in the main memory page fault occurs. The requested page has to be fetched from the secondary memory based on the entries in the page table and replace an already existing page in the main memory. The criteria on which the page to be replaced is selected is what sets one page replacement algorithm from another.

When the page that was selected for replacement and was paged out, is referenced again, it has to read in from disk, and this requires for I/O completion. This process determines the quality of the page replacement algorithm: the lesser the time waiting for page-ins, the better is the algorithm.

A page replacement algorithm looks at the limited information about accessing the pages provided by hardware, and tries to select which pages should be replaced to minimize the total number of page misses, while balancing it with the costs of primary storage and processor time of the algorithm itself. There are many different page replacement algorithms. We evaluate an algorithm by running it on a particular string of memory reference and computing the number of page faults.

Types

There are 3 major types

  1. First In First Out (FIFO)
  2. Least Recently Used (LRU)
  3. Optimal Page Replacement (OPR)
    4.

First In First Out

Description

This is the simplest page replacement algorithm. In this algorithm, operating system keeps track of all pages in the memory in a FIFO type queue. When a page needs to be replaced, the page in the front of the queue is replaced.

Advantages

  1. Easy to understand and execute.
  2. No overhead in choosing the page to replace.

Disadvantages

  1. Least efficient page replacement algorithm on an average.
  2. System needs to keep track of each frame
  3. Sometimes it behaves abnormally. This behaviour is called Belady’s anomaly
  4. Bad replacement choice increases the page fault rate and slow process execution

Least Recently Used

Description

The Least Recently Used (LRU) algorithm replaces the page that has not been called for by the CPU for the longest period of time. It is based on the observation that pages that have not been used for long time will probably remain unused for the longest time and are the best candidates for replacement.

Advantages

  1. LRU page replacement algorithm is quiet efficient.
  2. It does not suffer from Belady's Anomaly.

Disadvantages

  1. Comparitively higher difficulty.
  2. Its implementation may require substantial hardware assistance.

Optimal Replacement Algorithm

Description

This algorithm replace the page that will be used the least number of times in the future. This algorithm needs to know the full reference stream from the beginning to make precise replacements. This algorithm is better than LRU and is twice as good as FIFO.

Advantages

  1. Lowest page fault rate.
  2. Never suffers from Belady’s anomaly.
  3. Twice as good as FIFO Page Replacement Algorithm.

Disadvantages

  1. Needs to have the full reference stream as an input.
  2. Implementation requires substantial hardware assistance.
